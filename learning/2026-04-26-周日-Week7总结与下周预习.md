### 3.2 React vs Vue 3 对比总结

这是我今天重点思考的内容，把两个框架的异同整理了一下：

| 特性 | React | Vue 3 |
|------|-------|-------|
| **核心思想** | 单向数据流、All in JS | 渐进式、Vuefile 含 HTML/CSS/JS |
| **状态定义** | useState + useRef | ref/reactive |
| **模板** | JSX (JavaScript) | SFC `<template>` |
| **更新方式** | setState 触发重渲染 | Proxy 自动追踪 |
| **组件通信** | Props 向下、Context 跨级 | Props + Provide/Inject |
| **生命周期** | useEffect 模拟 | onMounted 等 |
| **路由** | React Router 6 | Vue Router 4 |
| **状态管理** | Redux/Zustand/Context | Pinia |
| **Hooks** | useXxx | Composables |
| **计算属性** | useMemo | computed |

**React 给我印象深刻的特点：**
1. JSX 很灵活，可以直接在模板里写 JavaScript 逻辑
2. Hooks 模式统一了类组件和函数组件
3. 但 useEffect 的依赖数组很容易出错，要小心

**Vue 3 给我印象深刻的特点：**
1. template 模板语法更接近原生 HTML，好上手
2. 响应式系统基于 Proxy，修改值自动追踪
3. 单文件组件（SFC）组织清晰

## 4. 学习内容

### 4.1 React 核心概念总结

**React 三原则：**
- 组件化：一切皆组件
- 声明式：描述 UI 状态，React 自动更新
- 单向数据流：数据从父组件流向子组件

**JSX 关键点：**
```jsx
// className vs class
<div className="container">

// 驼峰属性
<button onClick={handleClick}>
<input onChange={e => console.log(e.target.value)}>

// 条件渲染
{condition && <Component />}
{condition ? <True /> : <False />}
```

**Hooks 规则：**
- 只能在函数组件或自定义 Hook 里调用
- 只能在顶层调用，不能在循环、条件、嵌套函数里

### 4.2 useState 深入理解

```jsx
// 基本用法
const [count, setCount] = useState(0);

// 函数式更新（重要！）
setCount(prev => prev + 1);

// 惰性初始化
const [data, setData] = useState(() => {
  return JSON.parse(localStorage.getItem('data') || '[]')
})
```

### 4.3 useEffect 执行时机

```jsx
// 每次渲染后执行
useEffect(() => {
  document.title = `${count} times`
})

// 挂载时执行 + 卸载时清理
useEffect(() => {
  const timer = setInterval(() => setCount(c => c + 1), 1000)
  return () => clearInterval(timer)
}, [])

// 依赖数组
useEffect(() => {}, [])      // 只执行一次（挂载时）
useEffect(() => {}, [dep])  // dep 变化时执行
```

### 4.4 useRef vs useMemo vs useCallback

| Hook | 用途 | 场景 |
|------|------|------|
| useRef | DOM 引用、保持可变值 | 定时器、输入框、Canvas |
| useMemo | 缓存计算结果 | 复杂计算、避免重复计算 |
| useCallback | 缓存函数引用 | 传给子组件的回调、依赖数组 |

```jsx
// useMemo
const expensive = useMemo(() => computeExpensive(a, b), [a, b])

// useCallback
const handleClick = useCallback(() => {
  setCount(c => c + 1)
}, []) // 空依赖，函数引用保持不变
```

## 6. 心得感悟

今天是 Week 7 的最后一天，进行总结和下周预习。

**Week 7 的核心收获：**

1. **React 的思维模式**：从命令式（操作 DOM）转到声明式（描述状态），这个转变很重要。我以前写 jQuery 习惯了，现在要适应新的思考方式。

2. **Hooks 是 React 的精髓**：useState、useEffect、useRef 这些 Hook 把以前类组件的功能都搬到了函数组件里。Hooks 的规则要牢记：只能在顶层调用！

3. **React vs Vue 对比**：两个框架各有优势。React 更灵活（JSX 就是 JavaScript），Vue 更直观（模板语法）。作为前端开发者，两个都要掌握。

4. **性能优化要谨慎**：useMemo 和 useCallback 不是万能的，不要过早优化。先用 React DevTools Profiling 找到真正的性能问题。

**对学习的思考：**
