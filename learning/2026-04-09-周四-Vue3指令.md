## 学习过程

### 1. 条件渲染 v-if/v-else
- v-if：真正的条件渲染，会销毁/重建 DOM
- v-else：必须紧跟在 v-if 后面
- v-else-if：多条件判断
- v-show：只是切换 CSS 的 display 属性，不销毁 DOM

### 2. 列表渲染 v-for
- 遍历数组：`v-for="item in items"`
- 遍历对象：`v-for="(value, key) in object"`
- 索引：`v-for="(item, index) in items"`
- **重要**：必须使用 :key 绑定唯一标识，提高渲染性能

### 3. 双向绑定 v-model
- 原理：v-bind 绑定 value + v-on 监听 input 事件
- 修饰符：
  - .lazy：失去焦点时更新
  - .number：自动转为数字
  - .trim：自动去除首尾空格
- 适用元素：input、textarea、select、checkbox、radio

## 学习内容

### v-if vs v-show
| 特性 | v-if | v-show |
|------|------|--------|
| 原理 | 销毁/重建 DOM | 切换 display |
| 开销 | 高 | 低 |
| 适用 | 条件不常变化 | 频繁切换 |

### v-for 中的 key
- 必须使用唯一标识作为 key
- 不推荐使用 index 作为 key（会导致渲染问题）
- key 帮助 Vue 高效更新 DOM

### v-model 原理
```javascript
// v-model 本质上是这样工作的：
<input :value="message" @input="message = $event.target.value">

// v-model 简化了写法
<input v-model="message">
```

## 心得感悟

今天学习了 Vue 3 最常用的三个指令，深刻体会到：

1. **v-if 的销毁机制**：之前以为 v-show 更好用（不销毁 DOM），但后来发现 v-if 在条件不常变化时更合适，因为不会占用 DOM 节点。关键是要根据场景选择。

2. **v-model 的强大**：以前觉得双向绑定很简单，今天深入学习才发现它支持这么多修饰符（.lazy、.number、.trim），而且可以轻松处理复选框、单选框、下拉框等多种表单元素。

3. **key 的重要性**：虽然之前知道要用 key，但今天通过实践更深刻地理解到 key 帮助 Vue 识别每个节点的唯一性，特别是在列表数据变化时能高效更新。

4. **计算属性 computed**：在实现购物车总价时，使用 computed 比 methods 更合适，因为计算属性有缓存机制，依赖不变时不会重新计算。
