## 学习过程

### 1. 官方文档学习

今天主要学习了 Vue 3 官方文档中的 Slots 章节：

- **插槽基础**：理解 `<slot>` 元素作为内容出口，父组件提供内容
- **渲染作用域**：插槽内容在父组件作用域中定义，但可以访问父组件数据
- **默认内容**：插槽可以提供后备默认内容
- **具名插槽**：多个插槽时使用 `name` 属性区分，`v-slot` 指令或 `#` 简写
- **作用域插槽**：允许子组件向父组件暴露数据

### 2. 核心概念理解

#### 插槽 vs 函数参数

把插槽想象成 JavaScript 函数更易理解：
```javascript
// 父组件传递插槽内容
FancyButton('Click me!')

// 子组件渲染插槽内容
function FancyButton(slotContent) {
  return `<button>${slotContent}</button>`
}
```

#### 渲染作用域规则

- 插槽内容在**父组件**作用域中定义
- 插槽内容可以访问**父组件**的数据
- 插槽内容**不能直接访问**子组件的数据
- 这与 JavaScript 的词法作用域一致

## 学习内容

### 插槽核心知识点

| 类型 | 用法 | 场景 |
|------|------|------|
| 默认插槽 | `<slot></slot>` | 单一内容分发 |
| 具名插槽 | `<slot name="header">` | 多内容区域 |
| 作用域插槽 | `<slot :item="item">` | 子组件暴露数据 |
| 默认内容 | `<slot>后备内容</slot>` | 提供默认值 |

### v-slot 指令

```vue
<!-- 完整写法 -->
<ChildComponent>
  <template v-slot:header>...</template>
</ChildComponent>

<!-- 简写 (推荐) -->
<ChildComponent>
  <template #header>...</template>
</ChildComponent>

<!-- 默认插槽简写 -->
<ChildComponent>...</ChildComponent>
```

### 作用域插槽语法

```vue
<!-- 子组件 -->
<slot :user="user" :count="count"></slot>

<!-- 父组件使用 -->
<MyComponent v-slot="{ user, count }">
  {{ user.name }} - {{ count }}
</MyComponent>
```

## 心得感悟

今天学习 Vue 3 插槽，让我对组件化开发有了更深的理解。之前学习组件通信时，主要关注 props 和 emits，忽略了插槽这个强大的功能。

**最大的收获**：
1. 插槽本质上是**内容分发**机制，让父组件可以自定义子组件的显示内容
2. 作用域插槽是一个**很优雅的模式**，让子组件暴露数据给父组件，这在做列表渲染、表格等场景特别有用
3. 插槽和函数参数 analogy 非常形象，帮助理解其本质

**与 React 的对比**：
- React 的 children 类似 Vue 的默认插槽
- React 的 render props 类似 Vue 的作用域插槽
- Vue 的具名插槽比 React 更直观

**实际应用场景**：
- 弹窗组件的 header/body/footer
- 卡片组件的 header/slot/default
- 列表组件的每一项自定义渲染
