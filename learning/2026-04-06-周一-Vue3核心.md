### 2. 核心概念学习

#### Vue 3 简介
- Vue 3 是当前最新版本，采用 Composition API 作为核心
- 支持 Single-File Component (.vue 文件)
- 配合 Vite 作为构建工具，体验更佳

#### Composition API 核心
Composition API 是一组函数式 API，允许我们通过导入函数而不是声明选项来编写 Vue 组件。

**核心 API:**
- `ref()` - 创建响应式引用（适合原始类型）
- `reactive()` - 创建响应式对象（适合对象类型）
- `onMounted()` - 生命周期钩子
- `computed()` - 计算属性
- `watch()` - 监听器

#### ref vs reactive 区别
```javascript
// ref - 适合原始类型 (string, number, boolean)
// 访问需要 .value
import { ref } from 'vue'
const count = ref(0)
count.value++  // 需要 .value 访问

// reactive - 适合对象类型
// 直接访问属性
import { reactive } from 'vue'
const state = reactive({
  count: 0,
  name: 'Vue'
})
state.count++  // 直接访问
```

#### `<script setup>` 语法
这是 Vue 3 推荐的文件写法，编译时会更高效：
```vue
<script setup>
import { ref, onMounted } from 'vue'

const count = ref(0)

function increment() {
  count.value++
}

onMounted(() => {
  console.log('组件挂载完成')
})
</script>

<template>
  <button @click="increment">Count: {{ count }}</button>
</template>
```

#### Composition API 优势
1. **更好的逻辑复用** - 通过 Composable 函数复用逻辑
2. **更灵活的组织方式** - 相关逻辑放在一起
3. **更好的 TypeScript 支持** - 类型推断更自然
4. **更小的打包体积** - 模板编译更高效

## 💡 心得感悟

今天开始学习 Vue 3，之前用 Vue 2写过一些项目，但 Composition API 确实是新鲜东西。

最大的感受是：**这才是现代前端的写法！**

之前用 Options API 的时候，一个复杂组件的 data、methods、computed、watch 散落在不同地方，后期维护真的很痛苦。Composition API 让我可以把相关的逻辑放在一起，这种"关注点分离"的思路很棒。

另外，用函数式的方式写组件，TypeScript 的类型推断也更自然。之前为了类型定义要写一堆泛型，现在看起来和普通 JS 差别不大，这对正在学习 TS 的我很友好。

不过 ref 和 reactive 的选择确实需要注意点，之后要在实践中多体会。
