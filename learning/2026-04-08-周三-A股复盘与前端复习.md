#### 2.1 Composition API 基础

```typescript
// 基本的 Vue 3 组件结构
<script setup>
import { ref, reactive, computed, onMounted } from 'vue'

// 响应式数据
const count = ref(0)
const state = reactive({
  name: 'Vue 3',
  version: '3.3+'
})

// 计算属性
const doubled = computed(() => count.value * 2)

// 生命周期
onMounted(() => {
  console.log('组件已挂载')
})
</script>
```

#### 2.2 ref vs reactive 深入理解

| 对比项 | ref | reactive |
|--------|-----|----------|
| 适用类型 | 任意类型（推荐原始值） | 对象/数组 |
| 访问方式 | count.value | state.name |
| 模板自动解包 | ✅ | ✅ |
| 重新赋值 | 支持 | 不支持（替换整个对象） |
| 深层响应 | 默认深层 | 默认深层 |

#### 2.3 响应式原理（底层）

Vue 3 使用 **Proxy** 实现响应式：

```javascript
// 简化的响应式原理
const reactive = (target) => {
  return new Proxy(target, {
    get(target, key, receiver) {
      // 依赖收集
      track(target, key)
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      // 触发更新
      trigger(target, key)
      return Reflect.set(target, key, value, receiver)
    }
  })
}
```

**关键点**：
- track: 收集依赖（effect）
- trigger: 触发更新
- ref 内部本质也是调用 reactive

#### 2.4 常见问题与解答

**Q: 什么时候用 ref？什么时候用 reactive？**

A:
- 基本类型（string, number, boolean）→ ref
- 对象/数组 → reactive
- 总结：ref 更通用，reactive 更直观

**Q: 解构后失去响应性怎么办？**

A: 使用 toRef/toRefs

```typescript
import { reactive, toRef, toRefs } from 'vue'

const state = reactive({ a: 1, b: 2 })

// toRef: 保持单个属性响应性
const a = toRef(state, 'a')

// toRefs: 保持整个对象响应性
const { a, b } = toRefs(state)
```

**Q: shallowRef 和 shallowReactive 什么时候用？**

A: 大数据量或性能敏感场景，只做浅层响应

```typescript
const shallow = shallowRef({ deep: 'data' }) // 只监听 .value 变化
```
