### 上午学习（9:00 - 11:00）

今天主要学习 Vue 3 的官方状态管理库 Pinia。Pinia 是 Vue 官方推荐的新一代状态管理方案，Vuex 的作者也推荐使用 Pinia。

#### 1. Pinia 基础概念

Pinia 三大核心概念：
- **State**：状态，类似 Vuex 的 state
- **Getters**：计算属性，类似 Vuex 的 getters
- **Actions**： actions，类似 Vuex 的 actions/mutations

#### 2. Pinia 与 Vuex 的区别

| 特性 | Pinia | Vuex |
|------|-------|-----|
| API | 简单直接 | 复杂（mutation/action） |
| TypeScript | 原生支持 | 需要额外配置 |
| 响应式 | 自动 reactive | 需要手动 mapState |
| 模块化 | 扁平化，更直观 | 需要手动 namespaced |
| 调试 | 支持 Vue DevTools | 支持 Vue DevTools |

#### 3. Pinia 基本用法

```javascript
// stores/counter.js
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  // state: 返回对象的函数
  state: () => ({
    count: 0,
    name: '夜墨'
  }),
  
  // getters: 计算属性
  getters: {
    doubleCount: (state) => state.count * 2,
    // 访问其他 store 的 getter
    upperName: (state) => state.name.toUpperCase()
  },
  
  // actions: 方法
  actions: {
    increment() {
      this.count++
    },
    async fetchData() {
      // 支持 async/await
      const res = await fetch('/api/data')
      return res.json()
    }
  }
})
```

#### 4. 在组件中使用

```vue
<script setup>
import { useCounterStore } from '@/stores/counter'

const store = useCounterStore()

// 访问 state
console.log(store.count)

// 调用 action
store.increment()

// 响应式 - 直接解构会丢失响应式
const { count, name } = store // ❌ 错误
const count = computed(() => store.count) // ✅ 正确
</script>
```

## 学习内容

### Pinia 核心要点

1. **defineStore**：定义状态仓库
   - 第一个参数：唯一 ID
   - 第二个参数：选项对象（state/getters/actions）

2. **State**：
   - 必须是函数（支持 SSR）
   - 自动响应式

3. **Getters**：
   - 类似 computed
   - 可以返回函数
   - 可以访问其他 store

4. **Actions**：
   - 支持同步和异步
   - `this` 指向 store 实例

### Pinia 插件系统

Pinia 支持插件扩展：
- 持久化存储
- 打印日志
- 等等

### 响应式原理

Pinia 基于 Vue 3 的 Proxy 实现响应式：
- `reactive()` 处理 state
- `computed()` 处理 getters
- 不需要手动触发更新

## 心得感悟

今天学习了 Pinia，感受到 Vue 3 的生态确实比 Vue 2 简洁很多。Pinia 的 API 设计非常人性化，不需要像 Vuex 那样区分 mutation 和 action，直接在 actions 中修改 state 即可。

最大的感悟是：**Pinia 让状态管理变得简单而直观**。这符合 Vue 3 的设计理念 - 渐进式、高效、简洁。

对比之前学习的 Vuex，Pinia 的优势：
1. 更轻量（约 1KB vs 16KB）
2. TypeScript 原生支持
3. API 更简洁
4. 支持组合式 API 风格
