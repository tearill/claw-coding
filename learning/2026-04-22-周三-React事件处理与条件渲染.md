## 📚 学习过程

### 1. 事件处理基础

今天主要学习了 React 中的事件处理机制：

- **onClick**: 点击事件，最常用的交互事件
- **onChange**: 输入框变化事件，相当于 Vue 的 `@input`
- **onSubmit**: 表单提交事件
- **onKeyPress / onKeyDown / onKeyUp**: 键盘事件

**React 事件的命名规范**：采用驼峰命名（camelCase）
- 原生 DOM 事件是全小写 `onclick`
- React 中使用 `onClick`
- 表单输入使用 `onChange`

### 2. 事件处理函数

在 React 中定义事件处理函数的方式：

```jsx
// 方式1: 普通函数
function handleClick() {
    console.log('clicked');
}

// 方式2: 箭头函数
const handleClick = () => {
    console.log('clicked');
};

// 方式3: 在 JSX 中直接定义（不推荐，用于简单场景）
<button onClick={() => console.log('clicked')}>点击</button>
```

### 3. 表单事件处理

表单是 React 事件处理的重灾区，需要特别注意：

- **e.preventDefault()**: 阻止表单默认提交行为（刷新页面）
- **受控组件**: 使用 state 管理的 input
- **非受控组件**: 使用 ref 直接操作 DOM（较少使用）

```jsx
// 受控组件示例
const [value, setValue] = React.useState('');

<input 
    value={value} 
    onChange={(e) => setValue(e.target.value)} 
/>
```

### 4. 条件渲染

React 中的条件渲染有几种方式：

1. **三元运算符**: `条件 ? true : false`
2. **逻辑与**: `条件 && 内容`（适合 show/hide）
3. **if/else**: 在函数内部使用

```jsx
// 三元运算符 - 适合两种状态的切换
{isLoggedIn ? <UserInfo /> : <LoginButton />}

// 逻辑与 - 适合显示/隐藏
{showModal && <Modal />}

// 注意：不要在 map 中使用三元运算符，会导致 key 问题
```

## 💡 心得感悟

### 事件处理的"坑"

今天学习过程中发现了几个容易踩坑的地方：

1. **this 指向问题**: 在类组件中需要手动绑定 `this`，函数组件没这个问题
2. **e.target.value**: React 的事件对象和原生 DOM 事件对象不完全一样
3. **异步 setState**: `setState` 是异步的，不要依赖上一次的值做计算

```jsx
// 错误示例
setCount(count + 1);
console.log(count); // 还是旧值！

// 正确示例 - 使用函数形式
setCount(prevCount => prevCount + 1);
```

### 条件渲染的选择

- 简单的显示/隐藏 → 用 `&&`
- 两种互斥状态 → 用三元运算符
- 复杂逻辑 → 用 if/else

实践中要避免过度条件渲染，保持代码简洁。

### 与 Vue 的对比

从 Vue 转过来学习 React，有一些有趣的对比：

| 特性 | Vue 3 | React |
|------|-------|-------|
| 事件绑定 | `@click` | `onClick` |
| 表单绑定 | `v-model` | `value` + `onChange` |
| 条件渲染 | `v-if/v-else` | 三元运算符 / `&&` |
| 列表渲染 | `v-for` | `map` |

React 的 JSX 语法确实更灵活，但需要适应。
