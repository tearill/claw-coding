## 学习过程

### 1. DOM 基础概念
今天学习了文档对象模型（DOM）的核心概念。DOM 是浏览器将 HTML 文档解析为树形结构的数据结构，每个 HTML 标签都是树中的一个节点。

**核心理解**：
- DOM 树：文档节点 → 元素节点 → 文本节点/属性节点
- window 和 document 对象的关系
- 节点类型：Element（1）、Text（3）、Attribute（2）

### 2. 获取 DOM 元素
学习了多种获取元素的方式：

```javascript
// 现代方法（推荐）
document.querySelector('.class')      // 返回第一个匹配元素
document.querySelectorAll('.class')   // 返回 NodeList
document.getElementById('id')          // 快速获取 ID

// 传统方法
document.getElementsByClassName('class')
document.getElementsByTagName('div')

// 关系获取
element.parentNode
element.children
element.nextElementSibling
```

**实践**：在控制台中尝试获取页面元素，观察返回值类型。

### 3. 操作 DOM 内容与属性
学习了修改元素内容的几种方式：

```javascript
// 内容操作
element.innerHTML = '<p>HTML字符串</p>'  // 可解析 HTML
element.textContent = '纯文本'            // 安全，防止 XSS
element.innerText = '显示的文本'          // 考虑 CSS 样式

// 属性操作
element.setAttribute('data-id', '123')
element.getAttribute('data-id')
element.removeAttribute('data-id')

// 样式操作
element.style.color = 'red'
element.style.display = 'none'
element.classList.add('active')
element.classList.remove('active')
element.classList.toggle('active')
```

**重要认知**：innerHTML 有安全风险，textContent 更安全。

### 4. 创建与删除元素
```javascript
// 创建元素
const newDiv = document.createElement('div')
newDiv.textContent = '新元素'
document.body.appendChild(newDiv)

// 插入元素
parent.insertBefore(newElement, existingElement)
// 或使用 after(), before(), append(), prepend()

// 删除元素
element.remove()           // 现代方法
parent.removeChild(element) // 传统方法
```

### 5. 事件处理机制
这是今天的重点内容！

#### 5.1 事件监听
```javascript
element.addEventListener('click', function(event) {
  console.log('点击了！')
})

// 箭头函数
element.addEventListener('click', (e) => {
  console.log(e.target)
})

// 第三个参数：是否在捕获阶段触发
element.addEventListener('click', handler, false) // 默认，冒泡阶段
```

#### 5.2 事件对象
```javascript
element.addEventListener('click', (e) => {
  e.target        // 触发事件的实际元素
  e.currentTarget // 绑定事件的元素（即 this）
  e.preventDefault()  // 阻止默认行为
  e.stopPropagation() // 停止事件传播
})
```

#### 5.3 事件冒泡与捕获
- **捕获阶段**：从根节点向下传递到目标元素
- **目标阶段**：到达触发事件的元素
- **冒泡阶段**：从目标元素向上回传到根节点

```javascript
// 阻止冒泡
child.addEventListener('click', (e) => {
  e.stopPropagation()
})
```

#### 5.4 事件委托（重要！）
这是性能优化的关键技术：

```javascript
// 不推荐：为每个子元素单独绑定事件
listItems.forEach(item => {
  item.addEventListener('click', handleClick)
})

// 推荐：将事件绑定到父元素
list.addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    handleClick(e.target)
  }
})
```

**优点**：
1. 减少内存占用（只绑定一个事件）
2. 动态添加的元素自动拥有事件
3. 代码更简洁

### 6. 常用事件类型
- **鼠标事件**：click, dblclick, mouseenter, mouseleave, mousemove
- **键盘事件**：keydown, keyup, keypress
- **表单事件**：submit, change, input, focus, blur
- **文档事件**：DOMContentLoaded, load, scroll, resize

## 心得感悟

今天的学习让我对 DOM 操作有了系统性的理解。之前写代码时总是边查边用，没有形成完整的知识体系。

**几点深刻体会**：

1. **事件委托真香**：之前给列表项绑定事件都是循环绑定，今天学会事件委托后，发现代码简洁了很多，而且新增的元素自动拥有事件，这正是 Vue/React 虚拟 DOM 思想的底层原理！

2. **innerHTML 的危险性**：之前随手用 innerHTML，今天才知道如果用户输入包含 `<script>` 标签会有 XSS 风险，以后要用 textContent。

3. **性能意识**：大量 DOM 操作会导致页面卡顿，这让我意识到为什么 Vue/React 要用虚拟 DOM - 直接操作真实 DOM 确实很"重"。

4. **事件冒泡的利弊**：可以善用冒泡（事件委托），也要防范冒泡（阻止传播），关键看场景。
