## 3. 学习过程

### 3.1 闭包复习

闭包是 JavaScript 最核心的概念之一，今天重新梳理了一遍：

**什么是闭包？**
> 闭包是指函数能够记住并访问其外部作用域的变量，即使该函数在其定义的作用域之外执行。

**形成闭包的三个条件：**
1. 嵌套函数（函数内部定义函数）
2. 内部函数引用外部函数的变量
3. 内部函数在外部函数外部被调用

**闭包的常见应用场景：**
- 私有变量/封装
- 函数工厂
- 防抖/节流
- 回调函数
- 异步编程中的变量保持

### 3.2 this 指向复习

this 指向是 JavaScript 面试的高频考点，今天系统复习了 4 种绑定规则：

| 绑定方式 | 场景 | this 指向 |
|----------|------|----------|
| 默认绑定 | 独立函数调用 | 全局对象（strict 下 undefined） |
| 隐式绑定 | obj.method() | obj 对象 |
| 显式绑定 | call/apply/bind | 指定的第一个参数 |
| new 绑定 | new Fn() | 新创建的对象 |

**箭头函数的特殊性：**
- 箭头函数没有自己的 this
- 箭头函数的 this 是**定义时**外层作用域的 this，不是调用时的 this

### 3.3 call/apply/bind 复习

| 方法 | 语法 | 执行方式 | 参数形式 |
|------|------|----------|----------|
| call | fn.call(thisArg, arg1, arg2...) | 立即执行 | 逐个传递 |
| apply | fn.apply(thisArg, [args]) | 立即执行 | 数组形式 |
| bind | fn.bind(thisArg, arg1, arg2...) | 返回新函数 | 逐个传递 |

**手动实现的关键：**
- 将函数设置为 context 的临时方法
- 利用隐式绑定改变 this
- 执行后删除临时方法

## 4. 学习内容（知识总结）

### 4.1 闭包知识点速记

```
闭包 = 函数 + 记住外部变量
形成条件 = 嵌套 + 引用外部变量 + 外部调用
应用场景 = 私有变量 | 函数工厂 | 定时器 | 回调
注意事项 = 内存泄漏（及时解除引用）
```

### 4.2 this 绑定速记

```
默认绑定 → 独立调用 → window/undefined
隐式绑定 → obj.foo() → obj
显式绑定 → foo.call(obj) → obj
new 绑定 → new Foo() → 新对象
箭头函数 → 定义时外层 this → 不变
```

### 4.3 call/apply/bind 速记

```
call   → 立即执行 → 参数逐个
apply  → 立即执行 → 参数数组
bind   → 返回新函数 → 可柯里化
```
