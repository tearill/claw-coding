# Day 1: HTML 语义化标签学习笔记

## 什么是语义化标签？

语义化标签（Semantic HTML）是指具有明确含义的 HTML 标签，它们能够清晰地表达内容的结构意义。

## 常用语义化标签

| 标签 | 含义 | 用途 |
|------|------|------|
| `<header>` | 头部区域 | 页面或区块的标题、导航 |
| `<nav>` | 导航区域 | 页面导航链接 |
| `<main>` | 主要内容 | 页面的主体内容 |
| `<article>` | 文章内容 | 独立的文章内容 |
| `<section>` | 区块 | 页面中的一个章节 |
| `<aside>` | 侧边栏 | 辅助信息 |
| `<footer>` | 底部区域 | 页脚信息 |
| `<figure>` | 媒体容器 | 图片、图表等 |
| `<figcaption>` | 媒体说明 | figure 的标题 |

## 语义化的好处

1. **可读性**：代码更易读、易维护
2. **SEO**：搜索引擎更好地理解页面结构
3. **无障碍**：屏幕阅读器更好地解析页面
4. **一致性**：团队协作更高效

## 今日练习

创建了一个**个人介绍页面**，使用了以下语义化标签：

- `<header>` - 页面头部（姓名、职位）
- `<main>` - 主要内容区域
- `<section>` - 各个内容区块（关于我、技能、项目、联系方式）
- `<article>` - 项目经验中的独立项目
- `<footer>` - 页脚

## 注意事项

- 不要滥用 `<div>`，优先使用语义化标签
- 一个页面只有一个 `<main>`
- `<article>` 应该是独立的、可分发的内容
