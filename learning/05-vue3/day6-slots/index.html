🐱 Vue 3 插槽 (Slots) 演示

1. 默认插槽 (Default Slot)

最基本的插槽用法，子组件提供插槽出口，父组件提供内容

// 子组件模板 <button class="btn btn-primary"> <slot>默认文本</slot> // 后备内容 </button> // 父组件使用 <MyButton>点击我</MyButton>

基础按钮

点击我 🐾

使用默认内容

2. 具名插槽 (Named Slots)

多个插槽时使用 name 属性区分，用 v-slot 或 # 简写

// 子组件 <div class="card-component"> <div class="card-header"><slot name="header"></slot></div> <div class="card-body"><slot></slot></div> <div class="card-footer"><slot name="footer"></slot></div> </div> // 父组件 <my-card> <template #header>标题</template> <template #default>内容主体</template> <template #footer>底部</template> </my-card>

3. 作用域插槽 (Scoped Slots)

子组件通过插槽向父组件暴露数据，父组件决定如何渲染

// 子组件 List <ul> <li v-for="item in items" :key="item.id"> <slot :item="item" :index="$index"></slot> </li> </ul> // 父组件 <my-list :items="products"> <template #default="{ item, index }"> {{ index + 1 }}. {{ item.name }} - ¥{{ item.price }} </template> </my-list>

产品列表 (自定义渲染)

用户列表 (另一种渲染)

4. 条件插槽 (Conditional Slots)

使用 $slots 判断是否有内容传入，动态显示不同样式

// 子组件 <div class="card" :class="{ 'has-header': $slots.header, 'has-footer': $slots.footer }"> <div v-if="$slots.header" class="card-header"> <slot name="header"></slot> </div> <div class="card-body"> <slot></slot> </div> <div v-if="$slots.footer" class="card-footer"> <slot name="footer"></slot> </div> </div>
这是一个完整的卡片，有 header 和 footer
简单卡片 - 只有默认内容

💡 插槽要点总结

  • 插槽内容在父组件作用域中定义
  • 插槽出口在子组件模板中
  • 作用域插槽允许子组件向父组件暴露数据
  • # 是 v-slot: 的简写
  • $slots 可用于判断插槽是否被使用
