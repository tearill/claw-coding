### 上午 10:00-12:00 Vue 3 核心复习

复习 Week 5 学习的 Vue 3 核心知识：

#### 1. Composition API 核心
- `setup()` 函数是 Vue 3 组件的入口
- 响应式系统基于 Proxy 实现
- 生命周期钩子需要在 setup 中导入使用

#### 2. 响应式系统 ref vs reactive
| 特性 | ref | reactive |
|-----|-----|----------|
| 适用类型 | 基本类型 | 对象/数组 |
| 访问方式 | .value | 直接访问 |
| 底层实现 | Object.defineProperty | Proxy |
| 替换/解构 | toRef/toRefs | 无需处理 |

#### 3. 指令系统
- **v-if/v-else**：条件渲染，DOM 级别的增删
- **v-for**：列表渲染，必须配合 `:key`
- **v-model**：双向绑定，支持 `.lazy`、`.number`、`.trim` 修饰符
- **computed**：计算属性，基于依赖自动缓存

#### 4. 组件通信
- **Props**：单向数据流，子组件不能修改
- **Emits**：子向父通信，触发事件
- **Provide/Inject**：跨层级通信，祖先向后代注入

#### 5. 状态管理 Pinia
- 定义 Store：`defineStore(id, setup)` 或 `defineStore(id, options)`
- state：响应式状态
- getters：计算属性
- actions：同步/异步方法

#### 6. 路由 Vue Router 4
- 动态路由：`:id` 路径参数
- 导航守卫：`beforeEach` 全局守卫
- 完整导航解析流程

## 学习内容

### Vue 3 响应式原理（重点）

```javascript
// Vue 3 响应式核心
const reactive = (target) => {
  return new Proxy(target, {
    get(target, key, receiver) {
      track(target, key) // 收集依赖
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      trigger(target, key) // 触发更新
      return Reflect.set(target, key, value, receiver)
    }
  })
}
```

- **Proxy vs Object.defineProperty**：Proxy 能监听新增/删除属性
- **依赖收集**：在 get 时收集，set 时触发
- **computed**：内部使用 effect，带缓存

### 组件通信模式

```
父组件
  ↓ props
子组件
  ↑ emits

祖先 → provide → 后代 (inject)
```
