### 2. 核心知识点整理

#### Props 声明方式

```vue
<script setup>
// 数组语法
defineProps(['foo', 'bar'])

// 对象语法（带类型）
defineProps({
  title: String,
  likes: Number
})

// TypeScript 类型声明
defineProps<{
  title?: string
  likes?: number
}>()
```

#### Props 传递细节

- **静态 vs 动态**：静态直接传值，动态用 v-bind (:)
- **不同值类型**：Number、Boolean、Array、Object
- **单向数据流**：props 只能读取，不能直接修改
- **Prop 验证**：可以指定类型、必填、默认值、自定义验证器

#### Emits 触发事件

```vue
<script setup>
// 声明 emits
const emit = defineEmits(['update', 'delete'])

// 在方法中触发
const handleClick = () => {
  emit('update', { id: 1, name: 'test' })
}
</script>
```

#### Provide/Inject 跨层级通信

```vue
// 父组件
<script setup>
import { provide } from 'vue'
provide('theme', 'dark')
</script>

// 子组件
<script setup>
import { inject } from 'vue'
const theme = inject('theme')
</script>
```

## 学习内容

### 1. Props 核心要点

| 特性 | 说明 |
|------|------|
| 单向数据流 | 父 → 子，props 只读 |
| 静态传递 | `<Child title="Hello" />` |
| 动态传递 | `<Child :title="msg" />` |
| 类型声明 | String, Number, Boolean, Array, Object |
| 验证规则 | required, default, validator |

### 2. Emits 核心要点

| 特性 | 说明 |
|------|------|
| 声明方式 | `defineEmits(['eventName'])` |
| 参数传递 | `emit('event', payload)` |
| TypeScript | `defineEmits<{ (e: 'update', data: any): void }>()` |
| 验证 | 可以在父组件验证 emit 参数 |

### 3. Provide/Inject 核心要点

| 特性 | 说明 |
|------|------|
| 跨层级 | 祖先 → 后代，不需要层层传递 |
| 响应式 | 配合 ref/reactive 实现响应式 |
| 注入默认值 | `inject('key', 'default value')` |
| 读写分离 | 父提供只读或可读写状态 |

### 关键代码示例

```vue
// Parent.vue
<script setup>
import { ref } from 'vue'
import Child from './Child.vue'

const message = ref('Hello from Parent')
const childData = ref('')

const handleChildUpdate = (data) => {
  childData.value = data
}
</script>

<template>
  <Child :message="message" @update="handleChildUpdate" />
</template>
```

```vue
// Child.vue
<script setup>
defineProps({
  message: {
    type: String,
    required: true
  }
})

const emit = defineEmits(['update'])

const sendToParent = () => {
  emit('update', 'Data from Child')
}
</script>

<template>
  <p>{{ message }}</p>
  <button @click="sendToParent">Send to Parent</button>
</template>
```

## 心得感悟

今天学习了 Vue 3 的组件通信机制，收益颇丰！🐈‍⬛

### 最大的收获

1. **理解单向数据流**：Vue 3 强制 props 只读，这让我意识到数据流动性的重要性。之前写代码时经常想直接在子组件修改 props，现在明白了应该 emit 事件让父组件处理。

2. **Emits 的类型安全**：TypeScript 配合 emits 可以实现完整的类型检查，再也不怕传错参数了。

3. **Provide/Inject 的优雅**：解决了多层组件通信的痛点，不需要逐层传递 props，尤其是在写组件库的时候特别有用。

### 实践体会

- 写代码时先想清楚数据流向：是从上往下还是从下往上？
- 简单的一层父子通信用 props + emits
- 多层嵌套用 provide/inject 更简洁
- 避免直接修改 props，即使看起来"有效"，也会破坏数据可预测性

## 问题与反思

### 遇到的问题

1. **解构 props 失去响应性**：一开始直接解构 `const { foo } = defineProps(['foo'])`，后来发现这样会失去响应式。解决方法是用 `props.foo` 或者在 3.5+ 版本可以使用解构。

2. **Boolean 类型转换**：传递 boolean 时需要注意，Vue 有特殊的类型转换规则，`disabled` 不传值默认是 true。
