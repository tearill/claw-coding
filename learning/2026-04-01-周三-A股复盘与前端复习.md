### 2. 前端复习：浏览器渲染过程 (Week 4 Day 1)

#### 关键知识点回顾

**浏览器渲染关键路径 (Critical Rendering Path)**：

```
URL → DNS解析 → TCP连接 → HTTP请求 → 响应
  ↓
HTML解析 → DOM树构建
  ↓
CSS解析 → CSSOM树构建
  ↓
Render Tree (渲染树) → Layout (布局) → Paint (绘制) → Composite (合成)
```

**核心概念**：

1. **DOM 树**：HTML 文档的对象表示
2. **CSSOM 树**：CSS 样式表的对象表示
3. **渲染树**：DOM + CSSOM 的结合，只包含可见节点
4. **布局 (Layout)**：计算每个元素的位置和大小
5. **绘制 (Paint)**：将元素绘制到图层
6. **合成 (Composite)**：将多个图层合成为最终页面

#### 重排 vs 重绘

| 概念 | 触发条件 | 性能影响 |
|------|----------|----------|
| **重排 (Reflow)** | 元素位置/大小改变 | 高性能开销 |
| **重绘 (Repaint)** | 样式改变但不影响布局 | 中等性能开销 |

**触发重排的操作**：
- 添加/删除 DOM 元素
- 元素位置、尺寸改变
- 浏览器窗口大小改变
- 获取 offsetWidth/offsetHeight 等布局属性

**优化建议**：
- 使用 transform 和 opacity 进行动画
- 避免频繁读取布局属性
- 使用 CSS 类批量修改样式

### 3. 前端复习：Event Loop 与任务队列 (Week 4 Day 2)

#### 关键知识点回顾

**Event Loop 执行流程**：

```
1. 执行同步代码（调用栈）
2. 清空微任务队列（Promise.then, queueMicrotask）
3. 执行宏任务队列中的一个任务（setTimeout, setInterval, I/O）
4. 再次清空微任务队列
5. 浏览器可能执行渲染
6. 重复步骤 3-5
```

**宏任务 vs 微任务**：

| 类别 | 示例 | 执行时机 |
|------|------|----------|
| 宏任务 | setTimeout, setInterval, UI渲染 | 每次一个 |
| 微任务 | Promise.then, MutationObserver, queueMicrotask | 清空队列 |

**async/await 原理**：
- async 函数返回 Promise
- await 暂停函数执行，等待 Promise resolve
- await 后的代码相当于 Promise.then 的回调，属于微任务

### 练习：浏览器渲染与 Event Loop 综合练习

```javascript
// 04-browser-principles/03-review/
// 文件：browser-review.js

// 练习1：理解渲染时机
console.log('1. 同步代码开始');

setTimeout(() => {
  console.log('2. 宏任务 setTimeout');
  // 此时页面可能会渲染
}, 0);

Promise.resolve().then(() => {
  console.log('3. 微任务 Promise');
});

requestAnimationFrame(() => {
  console.log('4. requestAnimationFrame');
});

console.log('5. 同步代码结束');

// 执行顺序：1 -> 5 -> 3 -> 4 -> 2
// 注意：requestAnimationFrame 在下次渲染前执行

// 练习2：理解重排与重绘
// 假设有如下代码：
const element = document.getElementById('box');

// 触发重排（读取布局属性）
const width = element.offsetWidth;

// 修改样式（触发重绘）
element.style.width = '100px';

// 使用 transform（只触发合成，不重排不重绘）
element.style.transform = 'translateX(50px)';

// 练习3：Event Loop 与渲染
async function renderTest() {
  console.log('A. async 开始');
  
  // await 之前的代码同步执行
  document.body.innerHTML = '<div>A</div>';
  
  // await 后的代码属于微任务
  await Promise.resolve();
  document.body.innerHTML = '<div>B</div>';
  
  // setTimeout 是宏任务
  setTimeout(() => {
    document.body.innerHTML = '<div>C</div>';
  }, 0);
  
  console.log('Z. async 结束');
}

renderTest();
```

## 🔍 深入理解

### 浏览器渲染与 Event Loop 的关系

1. **渲染时机**：浏览器在每次 Event Loop 的宏任务完成后、下一个宏任务开始前，可能会执行渲染（取决于是否发生了变化）

2. **requestAnimationFrame**：这个API在每次渲染前被调用，是执行动画的最佳时机

3. **setTimeout vs requestAnimationFrame**：
   - setTimeout：不确定何时执行，可能掉帧
   - requestAnimationFrame：与浏览器刷新率同步，更平滑

4. **Vue 3 nextTick 原理**：Vue 3 使用 Promise 微任务队列来实现 nextTick，确保在 DOM 更新后执行回调
