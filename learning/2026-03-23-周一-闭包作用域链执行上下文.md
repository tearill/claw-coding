## 3. 学习过程

### 3.1 闭包基础概念

今天主要通过 MDN 文档和《你不知道的 JavaScript》上卷来学习闭包。

闭包的定义：**函数能够记住并访问其外部作用域的变量**，即使该函数在其定义的作用域之外执行。

形成闭包的三个条件：
1. 嵌套函数（函数内部定义函数）
2. 内部函数引用外部函数的变量
3. 内部函数在外部函数外部被调用

### 3.2 作用域链

JavaScript 采用词法作用域（Lexical Scope），作用域在代码编写时就确定了。

```
全局作用域 → 函数作用域 → 块级作用域
```

查找变量时，从当前作用域向上查找，直到全局作用域。

### 3.3 执行上下文

执行上下文是 JavaScript 代码执行时的环境，包含：
- 当前执行的代码
- 变量的作用域
- 函数的调用栈

执行上下文分为：
1. **全局执行上下文**：程序启动时创建
2. **函数执行上下文**：每次函数调用时创建
3. **Eval 执行上下文**（较少使用）

执行上下文的工作流程：
1. 创建阶段：创建变量对象、确定 this 指向、建立作用域链
2. 执行阶段：执行代码、变量赋值、函数调用

## 4. 学习内容

### 4.1 闭包的核心原理

```javascript
function outer() {
  let count = 0;
  
  function inner() {
    count++;
    return count;
  }
  
  return inner;  // 返回内部函数，外部持有对 count 的引用
}

const counter = outer();  // counter 是一个闭包
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

**关键点**：虽然 `outer` 已经执行完毕，但 `inner` 仍然可以访问 `count`，因为形成了闭包。

### 4.2 闭包的常见应用

1. **私有变量**
```javascript
function createPerson(name) {
  let _name = name;  // 私有变量
  
  return {
    getName() { return _name; },
    setName(newName) { _name = newName; }
  };
}
```

2. **函数工厂**
```javascript
function createMultiplier(factor) {
  return function(num) {
    return num * factor;
  };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);
console.log(double(5)); // 10
console.log(triple(5)); // 15
```

3. **防抖/节流（初步了解）**
```javascript
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
```

### 4.3 闭包与内存泄漏

**注意**：闭包会导致外部函数的变量无法被垃圾回收，如果不需要了要手动解除引用。

```javascript
function leak() {
  const bigData = new Array(1000000).fill('x');
  
  return function() {
    console.log(bigData.length);
  };
}

// 解决方案：使用后手动解除
const fn = leak();
fn = null;  // 解除引用，帮助 GC
```

### 4.4 作用域链图解

```
执行上下文 A (global)
├── 变量: a = 1
└── 作用域链: [global]

执行上下文 B (function outer)
├── 变量: b = 2
├── 作用域链: [outer, global]
└── 内部函数 inner 的 [[Scope]] = [outer, global]

执行上下文 C (function inner)
├── 变量: c = 3
└── 作用域链: [inner, outer, global]
```

### 4.5 this 指向问题

闭包中的 this 指向需要注意：

```javascript
const obj = {
  name: 'obj',
  getName() {
    // 这里的 self 解决了 this 指向问题
    const self = this;
    const fn = function() {
      console.log(self.name);  // obj
    };
    return fn;
  }
};

// 现代方案：箭头函数
const obj2 = {
  name: 'obj2',
  getName() {
    return () => console.log(this.name);  // obj2
  }
};
```

## 6. 心得感悟

今天学习闭包，对 JavaScript 的理解又深入了一层。之前只是会用，现在终于明白为什么会出现"闭包"这个概念——它本质上是 JavaScript 词法作用域的必然产物。

最大的收获是理解了**执行上下文**的概念。JavaScript 代码执行时，会创建一个个执行上下文，这些上下文通过作用域链连接起来，形成了一个"作用域链"。闭包就是利用这个机制，让内部函数能够访问外部函数的变量。

还有一个重要的点是：**闭包不是故意设计出来的特性，而是词法作用域的自然结果**。理解了这一点，就明白为什么闭包会出现了。

面试中闭包是高频考点，今天的学习为面试打下了基础。接下来要继续学习 `this` 指向和 `call/apply/bind`，这也是 JavaScript 的难点。
