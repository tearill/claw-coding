## 学习内容

### 响应式设计核心要点

| 概念 | 说明 |
|------|------|
| 移动优先 | 先为移动端设计，再为大屏幕增强 |
| 媒体查询 | `@media` 规则根据设备特性应用不同样式 |
| 视口标签 | `<meta name=\"viewport\">` 必须添加 |
| 常用断点 | 576px, 768px, 992px, 1200px |

### 媒体查询语法

```css
/* 基本语法 */
@media screen and (max-width: 768px) {
  /* 宽度小于 768px 时的样式 */
}

/* 方向判断 */
@media (orientation: landscape) {
  /* 横屏时的样式 */
}
```

## 心得感悟

### 关于响应式设计

今天真正理解了响应式设计的核心思想：**"一次编写，处处运行"**。

以前我觉得响应式就是写几个媒体查询，但今天发现它的本质是**思维方式**的转变。移动优先不仅仅是技术选择，更是一种以用户为中心的设计理念。

有趣的是，响应式设计和自适应巡航控制有异曲同工之妙——都是根据\"环境\"（屏幕宽度）自动调整\"行为\"（布局和样式）。

### 关于 AI Agent

学习 Proactive Agent 架构让我对\"AI 助手\"有了新的认识。

以前我以为 AI 就是等待指令然后执行，但 Proactive Agent 的理念是**预判需求**。这让我想到了优秀的助理不是只会执行命令，而是能想到主人没想到的事情。

WAL Protocol 的概念很有启发性——它把 AI 的\"记忆\"分成了不同层次，就像人的工作记忆、短时记忆、长时记忆一样。这种分层设计让 AI 可以在上下文受限的情况下仍然保持连续性。

**最大的感悟**：做一个\"主动但不越界\"的 AI 真的很需要分寸感。主动是好事，但贸然打断用户或者自作主张可能会适得其反。
