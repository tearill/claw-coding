## 学习主题

Vue 3 响应式系统核心 - ref 与 reactive 的深度理解

2. 理解 Vue 3 响应式的底层原理（Proxy）
3. 能够根据场景选择合适的响应式 API

### 2. 官方文档学习

- 阅读 Vue 3 官方文档中关于响应式 API 的部分
- ref(): 处理基本类型，创建包含 .value 属性的响应式对象
- reactive(): 处理对象/数组，创建深度响应式的 Proxy

### 3. 核心区别

| 特性 | ref | reactive |
|------|-----|----------|
| 数据类型 | 任意类型（通常用于原始值） | 对象/数组 |
| 访问方式 | 需要 .value | 直接访问 |
| 重新赋值 | 支持（替换整个值） | 不支持（只能修改属性） |
| 解构 | 会失去响应性（需 toRef/toRefs） | 会失去响应性（需 toRefs） |

## 学习内容

### ref API

```typescript
import { ref } from 'vue'

// 创建
const count = ref(0)

// 读取/修改（需要 .value）
console.log(count.value)
count.value++
```

### reactive API

```typescript
import { reactive } from 'vue'

// 创建
const state = reactive({
  count: 0,
  name: 'Vue 3'
})

// 直接访问
console.log(state.count)
state.count++
```

### 底层原理

Vue 3 使用 Proxy 实现响应式：
- ref 在内部调用 reactive
- 组件中模板自动解包 ref.value

## 心得感悟

今天学习 ref/reactive 时，突然想到之前学 JavaScript 闭包时的场景。ref 就像是一个"盒子"，外面包了一层 Proxy 来监听访问；而 reactive 则是直接对对象进行代理。

最大的收获是理解了"何时用 ref、何时用 reactive"：
- 原始值（string/number/boolean）→ ref
- 对象/数组 → reactive

另外 ref 的 .value 在模板中会自动解包，这个细节之前一直没注意，今天终于搞清楚了！
