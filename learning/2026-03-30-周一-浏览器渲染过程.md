## 3. 学习过程

### 3.1 浏览器渲染流程概述

从在地址栏输入 URL 到页面呈现，浏览器经历了以下主要阶段：

1. **URL 解析** - 浏览器解析输入的 URL
2. **DNS 解析** - 将域名转换为 IP 地址
3. **TCP 连接** - 与服务器建立 TCP 连接（三次握手）
4. **发送 HTTP 请求** - 浏览器发送请求报文
5. **接收响应** - 服务器返回响应（HTML、CSS、JS、图片等）
6. **渲染阶段** - 这是核心部分

### 3.2 关键渲染路径（CRP）详解

#### 第一步：解析 HTML 构建 DOM 树

- 浏览器接收 HTML 文本
- 词法分析（Tokenizer）→ 语法分析（Parser）
- 构建 DOM（Document Object Model）树
- DOM 树包含所有 HTML 元素及其关系

#### 第二步：解析 CSS 构建 CSSOM 树

- 浏览器解析 CSS 样式
- 构建 CSSOM（CSS Object Model）树
- 处理内联样式、外联样式、inline 样式

#### 第三步：合并 DOM + CSSOM = Render Tree

- 将 DOM 树和 CSSOM 树合并
- 生成渲染树（Render Tree）
- 渲染树只包含可见节点

#### 第四步：布局（Layout/Reflow）

- 计算每个节点的位置和大小
- 从根节点开始递归计算
- 布局信息存储在布局树中

#### 第五步：绘制（Paint）

- 将渲染树转换为像素信息
- 分层绘制
- 生成图层信息

#### 第六步：合成（Composite）

- 将各图层合成为最终页面
- 显示在屏幕上

### 3.3 关键概念理解

#### 渲染树 vs DOM 树

- DOM 树：包含所有 HTML 元素
- CSSOM 树：包含所有 CSS 规则
- Render 树：只包含可见元素，且包含计算后的样式

#### 重排（Reflow）与重绘（Repaint）

- **重排（Reflow）**：布局改变，需要重新计算布局
  - 触发条件：改变元素尺寸、位置、添加/删除元素
  - 代价：高，性能开销大
- **重绘（Repaint）**：样式改变，只需重新绘制
  - 触发条件：改变颜色、背景等不影响布局的属性
  - 代价：中等

#### 图层（Layer）优化

- 某些元素会被提升为独立的合成层（Compositing Layer）
- 触发条件：transform、opacity、position: fixed 等
- 优点：减少重排影响范围
- 缺点：内存占用增加

## 4. 学习内容总结

| 阶段 | 产物 | 说明 |
|------|------|------|
| HTML 解析 | DOM 树 | 所有 HTML 元素 |
| CSS 解析 | CSSOM 树 | 所有 CSS 规则 |
| 合并 | Render 树 | 可见节点 + 计算样式 |
| 布局 | Layout 树 | 位置和尺寸信息 |
| 绘制 | 图层 | 像素信息 |
| 合成 | 页面 | 最终呈现 |

**性能优化核心思想**：
1. 减少渲染次数（减少重排/重绘）
2. 减少渲染阻塞（CSS 放在 head，JS 放在 body 底部）
3. 关键CSS内联，非关键CSS异步加载
