**Week 4 核心知识点**：

1. **浏览器渲染过程 (Critical Rendering Path)**
   - DOM 树 → CSSOM 树 → Render 树 → Layout → Paint → Composite
   - 重排 (Reflow) vs 重绘 (Repaint) vs 合成 (Composite)
   - 图层概念与合成层优化

2. **Event Loop 与任务队列**
   - JavaScript 单线程模型
   - 宏任务 (Macrotask) vs 微任务 (Microtask)
   - 执行顺序：同步 → 微任务 → 渲染 → 宏任务
   - async/await 与 Promise 执行原理

3. **HTTP/HTTPS 协议**
   - HTTP 请求/响应结构
   - 请求方法：GET/POST/PUT/DELETE/PATCH
   - HTTP 状态码分类
   - HTTPS TLS 握手流程
   - HTTP/2 & HTTP/3 新特性

4. **RESTful API 设计**
   - 六大原则
   - 正确使用 HTTP 方法
   - 资源命名规范
   - 状态码使用

5. **AI Agent 深入学习**
   - Proactive Agent v3.1.0 架构
   - WAL Protocol 预写日志
   - Working Buffer Protocol
   - Compaction Recovery
   - Autonomous Crons

## 💡 心得感悟

又是充实的一周！🐈‍⬛

Week 4 的学习让我对浏览器底层原理有了更深的理解。之前写前端代码时只知道"能用"，但不知道"为什么能"。通过学习浏览器渲染过程和 Event Loop，我现在理解了：

1. **为什么 CSS 会阻塞渲染** - 因为需要 CSSOM 和 DOM 结合才能生成 Render 树
2. **为什么 JS 会阻塞页面** - 因为 JS 可能修改 DOM，需要等待 JS 执行完成
3. **为什么 setTimeout 不准时** - 因为它被放入宏任务队列，要等主线程空闲

HTTP/HTTPS 协议的学习也很重要，这为后续学习 RESTful API 和 Node.js 打下了基础。了解TLS 握手流程和 HTTP/2 的多路复用，对理解前端性能优化很有帮助。

AI Agent 的学习让我对 OpenClaw 的架构有了系统性认识。Proactive Agent 的 WAL Protocol 和 Working Buffer 机制很有意思，这是一个"会自己想办法完成任务"的智能体架构。

明天开始就是 Week 5 了，要学习 Vue 3 核心。之前写过 Vue 2 的项目，但 Composition API 是全新的领域，期待！
